# Правила добавления макросов

1. В каждом модуле должна быть строка `Option Explicit`.
2. Публичные процедуры должны иметь уникальные имена.
3. Название модуля начинается с `mod`.
4. Универсальная процедура принимает диапазон или адрес через параметры.
5. Процедура с суффиксом `_Default` содержит конкретные рабочие диапазоны.
6. Нельзя использовать `Select`, `Selection` и `Activate`, кроме процедур Word, работающих с выделением.
7. Перед изменением `Application.ScreenUpdating`, `EnableEvents` или `Calculation` необходимо сохранить старое состояние и восстановить его при ошибке.
8. Для каждого нового алгоритма желательно добавить тест в `tests/modSelfTests.bas`.
9. Пути к программам, словарям и файлам не должны быть жёстко зашиты в нескольких местах.
10. Перед публикацией удалить персональные данные, пароли и токены.
