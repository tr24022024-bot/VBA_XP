# Установка модулей

## Кодировка исходников

Файлы репозитория хранятся в UTF-8, чтобы GitHub правильно показывал русский текст.
Редактор VBA не во всех версиях Office корректно импортирует UTF-8. Поэтому предпочтителен
автоматический импорт через `Import-VbaModules.ps1`: сценарий создаёт временные копии
в Windows-1251 и импортирует именно их.

## Ручной импорт в Excel

Перед ручным импортом сохраните `.bas` как `ANSI / Windows-1251`, если редактор VBA
искажает кириллицу.

1. Создайте или откройте книгу `.xlsm`.
2. Нажмите `Alt + F11`.
3. В редакторе VBA выберите нужный проект.
4. Выполните `File → Import File`.
5. Импортируйте файлы:
   - `src/excel/modCore.bas`
   - `src/excel/modLevenshtein.bas`
   - остальные необходимые модули.
6. Сохраните книгу.

## Первичная инициализация

После импорта выполните:

```vb
Repository_Initialize
```

Будут созданы служебные листы `КОДИРОВКА` и `СЛОВАРЬ`.

## Проверка

Импортируйте `tests/modSelfTests.bas` и выполните:

```vb
Repository_SelfTest
```

## Автоматический импорт

PowerShell-сценарий:

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\Import-VbaModules.ps1 `
  -WorkbookPath "C:\Work\Book.xlsm" `
  -SourceRoot ".\src\excel"
```

Для автоматического импорта требуется разрешение:

```text
Файл → Параметры → Центр управления безопасностью →
Параметры центра управления безопасностью →
Параметры макросов →
Доверять доступ к объектной модели проектов VBA
```

После импорта рекомендуется снова отключить этот параметр.

## Порядок импорта

Сначала импортируйте:

1. `modCore.bas`
2. `modLevenshtein.bas`
3. функциональные модули
4. тесты
